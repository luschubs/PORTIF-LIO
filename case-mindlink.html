A dashboard that helps HR teams act, not just read.

End-to-end product design for a psychosocial risk assessment platform — from brand identity to a fully functional, AI-prototyped React dashboard deployed for client demos.

The Problem

Compliance is the law. Action is the design challenge.

MindLink is a psychosocial risk assessment company that helps organizations comply with Brazil's updated NR-01 regulation which now requires mandatory mapping of psychosocial risks in the workplace, with enforcement beginning in 2026.

Their diagnostic process generates massive amounts of data: survey responses, risk classifications by department, participation rates, severity levels, root causes. But all of this was being delivered as static PDF reports. HR teams had no way to dynamically explore the data, compare departments, or track progress over time.

The challenge: transform a document-based delivery into a living, actionable dashboard.

Design Process

From zero brand to functional prototype.

01

Brand Identity

Started from zero. Designed MindLink's complete identity — the dashed infinity logo, deep teal palette, and typography pairing that conveys clinical credibility.

02

Information Architecture

Mapped 5 core screens: Overview, Department comparison, Detail, Action Plan (kanban), and Reports. Built around the questions HR actually asks.

03

UI in Figma

Light theme with dark teal sidebar — brand presence without overwhelming the data. Color-coded severity system (red/amber/green) for instant scannability.

Vibe Coding

Built a real working prototype with AI.

I took the Figma design further by using AI-assisted development (Claude) to translate it into a fully interactive React build. Real navigation, animated charts, a live kanban board, responsive states.

This wasn't a Figma prototype — it's a deployed web application I can hand to the client today. Hosted on GitHub Pages for stakeholder review.

Key Decisions

Three choices that shaped the product.

A

Action Plan as a first-class screen

Most dashboards stop at showing data. MindLink's connects insights to actions — a kanban board to track interventions from "Pending" to "Completed".

B

Severity as color, not just numbers

HR professionals aren't data analysts. A score of "8.2" means nothing without context — every number is paired with a traffic-light visual indicator.

C

AI as a design tool, not a shortcut

Used Claude to bridge static mockups and real interaction. The design decisions stayed mine — AI just compressed the build cycle.

Reflection

Showing data vs. helping someone decide.

This project taught me the difference between visualization and decision support. Every chart, every score card, every color choice exists to answer a question the HR team is already asking.

The biggest lesson: design systems for complex data need to be opinionated. Neutral visualization is the same as no visualization.

Next steps: usability testing with HR professionals, iterating on the Action Plan based on client feedback, and moving from prototype to production with a development team.
